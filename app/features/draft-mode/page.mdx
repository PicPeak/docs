# Draft Mode

Draft mode lets you prepare events fully before making them visible to clients.

## How it works

When you create a new event, you can save it as a **draft**. Drafts are:

- Not accessible via the gallery link (guests cannot view them).
- Fully editable --- upload photos, organize categories, configure settings.
- Visible only in the admin panel.

Once you are satisfied with the gallery, **publish** the event. Publishing makes it accessible to guests via the gallery link and can trigger the initial email notification.

## Workflow

1. Create a new event and leave it in draft state.
2. Upload and organize photos, set categories, configure themes.
3. Preview the gallery from the admin panel.
4. When ready, publish the event.
5. Optionally send the gallery link email to the customer.

## Unpublishing

Published events can be returned to draft state if you need to make changes before the client sees the updates. While in draft state, the gallery link returns an inaccessible response.
