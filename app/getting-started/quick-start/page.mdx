# Quick Start

Get PicPeak running in under five minutes with Docker.

**Requirements:** Docker v20.10+ and Docker Compose v2.0+. At least 2 CPU cores, **4 GB RAM** (2 GB is workable only with [`UPLOAD_PROCESSOR_CONCURRENCY=1`](/deployment/docker#low-memory-hosts) — sharp/libvips decode peaks past 1.5 GB on heavy upload batches and OOM-kills the backend on smaller hosts, surfacing as 503s on thumbnails), 20 GB storage.

<Steps>

### Clone and configure

```bash
git clone https://github.com/PicPeak/picpeak.git
cd picpeak
cp .env.example .env
```

Generate secure secrets and edit `.env`:

```bash
# Generate values
openssl rand -base64 64    # use for JWT_SECRET
openssl rand -base64 32 | tr -d '$'  # use for DB_PASSWORD
openssl rand -base64 32 | tr -d '$'  # use for REDIS_PASSWORD
```

Open `.env` and set these three values:

```env
JWT_SECRET=<paste generated value>
DB_PASSWORD=<paste generated value>
REDIS_PASSWORD=<paste generated value>
```

### Create storage directories

```bash
mkdir -p events/active events/archived data logs backup storage
```

The backend container will chown these to its internal `nodejs` user on first boot — no `PUID`/`PGID` setup needed. If you've pinned `user:` in a compose override, see [Permissions](/deployment/docker#permissions) for the manual pre-chown step.

### Start PicPeak

<Tabs items={['Production (recommended)', 'Development']}>
<Tabs.Tab>
```bash
docker compose -f docker-compose.production.yml pull
docker compose -f docker-compose.production.yml up -d
```
</Tabs.Tab>
<Tabs.Tab>
```bash
docker compose up -d
```
</Tabs.Tab>
</Tabs>

Wait for all containers to report as healthy:

```bash
docker compose ps
```

### Open PicPeak

Open `http://localhost:3000` in your browser. The admin panel is at `http://localhost:3000/admin`.

</Steps>

<Callout>
The frontend (port 3000) serves the UI and proxies API requests to the backend (port 3001). You don't need to access port 3001 directly.
</Callout>
