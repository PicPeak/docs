# Events API

## Create an event

```bash
API_URL="http://localhost:3001/api"
TOKEN="your-jwt-token"

curl --fail --silent --show-error \
  -X POST "$API_URL/admin/events" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "event_type": "wedding",
    "event_name": "Emily & Jordan Celebration",
    "event_date": "2025-06-07",
    "customer_name": "Emily Carter",
    "customer_email": "emily@example.com",
    "admin_email": "studio@example.com",
    "require_password": true,
    "password": "Shutter123",
    "expiration_days": 45
  }' | jq
```

A successful response includes `id`, `slug`, and `share_link`.

### Required fields

| Field | Type | Description |
|-------|------|-------------|
| `event_type` | string | Event category (e.g., `wedding`, `corporate`, `birthday`) |
| `event_name` | string | Display name for the gallery |
| `event_date` | string | ISO date (YYYY-MM-DD) |
| `customer_name` | string | Client name |
| `customer_email` | string | Client email address |

### Optional fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `admin_email` | string | --- | Photographer notification email |
| `require_password` | boolean | `false` | Require password for gallery access |
| `password` | string | --- | Gallery password |
| `expiration_days` | integer | `30` | Days until gallery expires |

## Resend gallery email

```bash
curl --fail --silent --show-error \
  -X POST "$API_URL/admin/events/$EVENT_ID/resend-email" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"password": "Shutter123"}' | jq
```

Omit `password` to send the standard security message instead of including credentials.

## Field naming

The API uses `customer_name` and `customer_email` fields. Legacy `host_*` fields are no longer accepted. Migrations copy legacy data automatically during upgrades.
